--->C’est quoi git?

   A) Git : est un logiciel de gestion de versions (Version Control System) qui suit l’évolution des fichiers sources et garde les anciennes versions de chacun d’eux sans rien écraser.
     ---->C’est quoi github?
   B) GitHub est une plateforme de "codes" open-source. Pour faire court, c'est une sorte de réseaux social de développeurs. 
     ---->Les commandes de git :
1- git config --global user. email sam@google.com:
    On l’utilise pour configurer les préférences de l’utilisateur : son mail, l’algorithme utilisé pour diff, le nom d’utilisateur et le format de fichier etc.
2-git  init :
  Cette commande est utilisée pour créer un nouveau dépôt GIT. 
3- git  add : 
   peut être utilisée pour ajouter des fichiers à l’index. 
4- git clone:
  est utilisée pour la vérification des dépôts.  Si le dépôt se trouve sur un serveur distant
5- git commit :
  permet de valider les modifications apportées au HEAD
6- git push origin master : 
  Un simple push envoie les modifications locales apportées à la branche principale associée
7- git checkout -b <nom-branche> : 
 peut être utilisée pour créer des branches ou pour basculer entre elles. 
8- git remote –v : 
  permet à un utilisateur de se connecter à un dépôt distant. La commande suivante répertorie les dépôts distants actuellement configurés.
9-Git pull : 
  Pour fusionner toutes les modifications présentes sur le dépôt distant dans le répertoire de travail local.
